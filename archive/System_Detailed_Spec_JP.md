# システム詳細仕様書: 日本株 売買シグナル基盤 (Japan Stock Signal Platform)

## 1. システム概要
日経225銘柄を対象とし、直近のニュースセンチメント、テクニカル指標、および開示情報（TDnet/EDINET）を複合的に評価して売買シグナルを判定・提示するプラットフォーム。フロントエンドおよびAPI・AI推論バックエンドを並行して構築し、半自動運用を実現する。

## 2. システムアーキテクチャ
### 2.1 フロントエンド
- **フレームワーク**: React 18 + Vite + TypeScript
- **スタイリング**: Vanilla CSS (プロトタイプ `japan-stock-real-link-dashboard.html` のCSS Variablesを活用したモダンなテーマ切替、Glassmorphism等の高級感あるUIを継承)
- **グラフ描画**: Chart.js (react-chartjs-2 等)
- **状態管理**: React Hooks (useContext) または Zustand

### 2.2 バックエンド・デプロイメントアーキテクチャ
- **本番環境(想定)**: 
  - **フロントエンド**: Cloudflare Pages 等の静的ホスティング（Edgeネットワークによる高速配信）。バックエンドとは分離。
  - **バックエンド**: 外部のPythonサーバー（Render, AWS EC2, GCP, VPS等）。推論・データ処理などの重いタスクを担当。
- **通信設計**: フロントエンドからバックエンドAPIへの通信は、Cloudflare経由または直接CORS許可の元で行う設計（環境変数 `VITE_API_BASE_URL` による動的ルーティング）。
- **フレームワーク**: Python 3.10+ & FastAPI
- **分析・推論AI**: OpenAI API / Gemini API 等のLLMを用いたニュース・開示資料のセンチメント分析（ポジネガ判定および点数化スコアリング）
- **データ取得元**:
  - 株価データ・テクニカル: yfinance または独自金融API
  - ニュース: News API, スクレイピングの併用
  - 開示情報: TDnet (JPX API/適時開示情報閲覧サービス), EDINET API
- **バックテストエンジン**: pandas / backtrader を用いたシミュレーションエンジン

## 3. 機能要件詳細

### 3.1 日経225銘柄検索・一覧表示 (Step 1)
- **対象**: 日経225採用銘柄の225件（最新版）。
- **ロジック**: フロントエンド側に初期表示用の軽量銘柄マスター(コード、社名、英語名、業種)を保持し、高速なインクリメンタルサーチ（リアルタイムフィルタリング）を実現。

### 3.2 個別銘柄詳細・シグナルダッシュボード (Step 2)
- **UI表示**: 選択中銘柄の名前、価格、前日比、判定KPIカード群、予測チャートカード等。
- **短期シグナル算出**: テクニカル（MA5/MA25, RSI等）分析と、AIによる直近短期ニュースのセンチメント複合評価。
- **長期シグナル算出**: AIを用いたTDnet/EDINET開示情報（決算短信・業績修正等）のファンダメンタルズ長期影響評価。
- **チャート機能**: 過去価格推移とMA、および推測AIによって計算された「短期トレンド予測ライン」の表示。

### 3.3 評価根拠（ニュース・開示資料）の詳細とリンク
- 分析に使用したニュース・開示資料のリスト化機能。
- 表示項目: スコア寄与度(+/-ポイント)、ソース種類(TDnet/IR/News等)、AIが算出した「評価理由要約」、直接アクセス可能な「元ソースURL」。

### 3.4 バックテスト・パフォーマンス表示 (Step 3)
- **評価結果画面**: 過去指定期間におけるシグナルの勝率、平均損益、最大ドローダウンなどの統計データ表示。
- **表示要件**: シミュレーションによる想定連敗数、プロフィットファクターなどをコンポーネントとして表示。

### 3.5 半自動運用・サマリー推論リスト (Step 4, 5)
- **トップ画面（総合ビュー）**: 市場全体でシステムが算出した「今日の買い候補 TOP10」「今日の売り候補 TOP10」一覧。
- **表示内容**: 銘柄情報と共に「想定保有日数」「想定損切り水準(-5%等)」「利確水準(+8%等)」を提示し、人間の最終判断を促す。

## 4. 非機能要件・制約
- **セキュリティの徹底**: APIキー（LLMやニュースAPI）や通信設定は `.env` などの環境変数で管理し、ソースコードには絶対にハードコードしない。
- **パフォーマンスと非同期処理**: バックエンドのAIスクレイピング通信は時間とコストがかかるため、非同期（Celery/BackgroundTasks＋Redis/DBキャッシュ）にて定期実行させる。フロント側のロードをブロックしないこと。
- **最新安定版の採用**: FastAPIやReact等の各ライブラリはすべて公式推奨の最新安定版 (Latest Stable) を使用すること。
- **分離アーキテクチャ (Cloudflare想定)**: Cloudflare等の静的ホスティングで動くフロントエンドから、外部のPython APIサーバーへ安全に通信するため、CORSポリシーの適切な設定とURLの環境変数管理を徹底する。

## 5. API エンドポイント設計 (FastAPI想定)
- `GET /api/stocks` : マスターリスト取得（※暫定フロント内保持の場合不要）
- `GET /api/stocks/{code}/summary` : 該当銘柄の現在価格・各種信号スコアを返却
- `GET /api/stocks/{code}/news` : 寄与した分析済みニュースリスト
- `GET /api/stocks/{code}/docs` : 寄与した分析済みTDnet/EDINET開示リスト
- `GET /api/stocks/{code}/backtest` : 指定モデルでのバックテスト戦績
- `GET /api/recommendations` : 当日の全体シグナル上位・下位一覧を返却
